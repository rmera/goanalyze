    Superimposing a trajectory
        • The reference
        • The target selection
        • The “moved” selection
      • An example

    You can superimpose a whole trajectory to a given reference. You need 3 selections:

    The reference

    For which the first state will be taken.

    The target selection

    This selection needs to have the same number of atoms as the reference. For each frame of this selection gomd will calculate the superposition to the first frame of the reference, thus obtaining rotation and translation matrices.

    The “moved” selection

    For each frame, the rotation + translation matrices obtained with the corresponding frame of the target will be applied to this selection. It needs to have the same amount of frames as the target selection.

    An example

    Say you want to translate/rotate your protein trajectory to match a reference, but you want to only consider the alpha carbons to calculate the superposition (of course, you want to translate/rotate your whole protein, not only the alpha carbons!).

    First, create a selection with only the alpha carbons of the reference:

    select alpha-ref, refProtein and name CA

    Now create a selection with the alpha carbons of the trajectory

    select alpha-traj, trajectory and name CA

    Now alpha-ref is your reference selection, alpha-traj is your target selection and trajectory (the object containing your trajectory) is the moved selection. You are ready now to call gomd:

    gomd super, alpha-ref, alpha-traj, trajectory

    A new, superimposed trajectory will appear, named “trajectory_sup”

    Note that this can use a lot of memory if your trajectory is large!
